Redis字符串背后的秘密

发表于 2018-08-15 | 分类于 Redis Study

一、前言

  Redis字符串是一个由字节数组组成的序列，其实和JAVA中的Map类结构很像，通过key去取值、存放值。在Redis中，字符串可以存储三种类型的值，字符串、整数、浮点数。用户可以通过给定的任意数值，对存储着整数和浮点数的字符串执行自增或者自减的操作，在Redis中可以明确的区分字符串、整数和浮点数的做法是一种优势，在存储上拥有灵活性。当然如果只用Redis中的字符串结构就如同一个memcache服务器一样。以下是存储一个字符串的示例:

二、命令及使用方式

  字符串拥有一些可以操作字符串、整数类型的字符串、浮点数类型的字符串的基础命令，其中包括GET、SET、DEL、INCR、DECR、INCRBY、DECRBY等基本的命令。可以看一下对应命令以及作用:

命令 使用方式 描述
GET get word 获取存储在key为word中的值
SET set word world 设置key为word的值为world
DEL del word 删除key为word中的值(这个命令对所有的类型都适用)
INCR incr word 将key为word存储的值加上1
DECR decr word 将key为word存储的值减去1
INCRBY incrby word 12 将key为word存储的值加上12
DECRBY decrby word 12 将key为word存储的值减去12
APPEND append word myWorld 将值追加到key为word的字符串的后面
GETRANGE getrance word 0 2 获取key为word，起始位置为0，终止位置为2的字符串，相当于截取操作，包括0也包括2
SETRANCE setrance word 2 xiaojie 将key为word的字符串从2开始位置往后设定为xiaojie
STRLEN strlen word 获取保存在key为word中的字符串的长度

这里需要注意的是如果是使用INCR、DECR、INCRBY、DECRBY等操作必须存储的数据为整数或者浮点数，不然Redis会报出错误。当用户将一个值存储到Redis字符串的时候，Redis会进行一次判断判断你存放的这个字符串是否可以被转换成整数或者浮点数，如果可以转换Redis就会允许用户使用自增自减等操作。
以上都是在Redis中常用以及需要去了解或者掌握的相关字符串的操作命令，其中APPEND往后的命令就趋向于JAVA中的对于字符串的操作差不多，可以通过redis-cli来自己验证一下上面的命令。

三、SDS简单动态字符串

  出于好奇心，想了解一下Redis内部是如何实现字符串命令的。在了解Redis对于字符串的保存机制之前先了解一种特殊的字符串:SDS(简单动态字符串)，当然Redis底层是通过C语言写的。所以看之前还是先复习一下以前学习的C语言基础吧！
  正题开始，SDS字符串在Redis中被广泛的使用，包括对于简单的保存字符串，或者是AOF持久化等都有SDS的面孔。首先我们看看Redis是如何定义SDS字符串的，在sds.h文件中定义了一个sdshdr结构声明:

struct sdshdr {

  //记录buf数组中已经使用的字节的数量
  //等于SDS字符串的长度
  int len;

  //记录buffer数组中未使用的字节的数量
  int free;

  //字节数组，用于保存字符串
  char buf[]
}
为什么使用SDS字符串呢，它与C字符串的区别又是哪些:
  1. 常数复杂度获取字符串的长度:C字符串并不记录自身的长度，所以为了获取字符串的长度必须遍历整个字符串显然这个复杂度就是O(N)效率就极为低下，而SDS字符串在内部本身就定义了长度的属性，所以无论获取多复杂的字符串的长度都是O(1)，这样的效率就会高很多，从这一点确定访问字符串不会成为Redis的性能瓶颈。
  2. 杜绝缓冲区溢出:什么是缓冲区溢出呢？缓冲区溢出就是当更改字符串的长度大于本身存储的长度的时候会占用其他的空间导致数据出现问题，这都是由于C字符串无法知道长度导致的，而SDS字符串完美的解决了这种情况，当改变字符串时首先会判断容量是否足够更改后的字符串，如果可以就添加否则会增加长度来满足改变后的字符长度。
  3. 减少修改字符串时带来的内存重分配次数:如果在Redis中使用C字符串那么每次字符串更改的时候都需要重新进行一次内存的分配来满足对字符串更改操作的需求，比如拼接字符串每次就必须为拼接后的字符串的重新分配一次内存空间否则就可能造成缓冲区溢出，比如缩短字符串操作，缩短完每次必须重新分配一次长度否则可能会造成内存泄露，而内存分配又是一个比较耗时的操作，因此每操作一次就为其分配内存会让Redis的效率大打折扣。而SDS字符串则为其做出了两种优化:
  (1). 空间预分配:空间预分配用于字符串的增长操作，当修改SDS字符串的时候，程序不仅为其分配所需要的空间同时为其分配额外的空间，在Redis中分配额外的大小的空间是由一些规定决定的。如果SDS进行修改后长度小于1024那么程序就分配和修改后字符串长度一样的额外空间。如果大于1024长度那么额外空间最大也就是1024长度。这个操作可以在一定程度上减少内存分配的策略。
  (2). 惰性空间释放:惰性空间释放用于SDS字符串缩短操作，当SDS字符串缩短时，程序并不会立刻执行内存分配操作将多余的空间进行回收，而是通过free属性用于记录多余的字符空间，方便以后需要对字符串进行增加操作的时候不需要再次进行内存重分配的操作也是一种减少程序重分配的策略。
  4. 二进制安全:C字符串中的字符必须符合某种编码，并且除了末尾以外不能存在空字符，否则读到空字符的位置就会被程序误以为已经读取到了字符的末端造成读取的数据不正确。因为这样的限制导致C字符串只能保存文本类型的数据而不能保存图片、音频等二进制数据。SDS字符串操作是二进制安全的，所有的SDS API都会以处理二进制的方式处理保存在SDS buf[]中的数据程序不会对其做任何的过滤转义等操作。数据存入是什么样的，那么读取的数据就什么样。
下面可以看一下大体上的C字符串与SDS字符串的区别:

C字符串 SDS字符串
获取字符串长度的复杂度为O(N) 获取字符串长度的复杂度为O(1)
不安全容易造成缓冲区溢出和泄漏 安全，不会造成缓冲区的泄漏和溢出
修改字符串的长度N次就必须执行内存分配N次 修改字符串长度N次，至多执行内存分配N次
只能保存文本数据(会对二进制数据中的特殊符号做处理) 二进制，文本都可以

四、Redis数据结构对象实现

Redis中统一用RedisObject这种结构代表保存的对象:

typedef struct redisObject {

  //类型  
  unsigned type:4;

  //编码
  unsigned encoding:4;

  //指向底层实现数据结构的指针
  void *ptr;

  //最后一次使用这个对象的时间
  unsigned lru:REDIS_LRU_BITS;

  //计数值
  int refcount;
}
type指定的是使用的对象的类型也就是Redis实现的五个(可以通过TYPE指令查看Redis中key的对象类型)，encoding指定的是对象的编码，数据的构成就是由这些编码实现的。下面是常见的对象编码表:

编码常量 编码所对应的底层数据结构
REDIS_ENCODING _INT long类型的整数
REDIS_ENCODING _EMBSTR embstr编码的简单动态字符串
REDIS_ENCODING _RAW 简单动态字符串
REDIS_ENCODING _HT 字典
REDIS_ENCODING _LINKEDLIST 双端链表
REDIS_ENCODING _ZIPLIST 压缩列表
REDIS_ENCODING _INTSET 整数集合
REDIS_ENCODING _SKIPLIST 跳跃表和字典

ptr就是对象的指针，指向了实现对象的结构类型。使用OBJECT ENCODING命令能看到key所对应的数据等编码实现。

五、Redis字符串命令底层实现

字符串的编码可以是int、raw或者embstr类型。具体的Redis字符串命令对应编码的实现可以看下表:

命令 int编码 embstr编码 raw编码
SET 使用int编码保存值 使用embstr编码保存值 使用raw编码保存值
GET 拷贝对象保存的整数值转换成字符串返回 直接返回字符串 直接返回字符串
APPEND 将对象转换成raw编码，然后按照raw处理方式处理 将对象转换成raw编码，然后按照raw处理方式处理 调用sdscatlen，将字符串拼接到尾部
INCRBY 对整数进行加法计算得到的结果以整数保存 不能执行会报错 不能执行会报错
DECRBY 对整数进行减法计算得到结果以整数保存 不能执行会报错 不能执行会报错
STRLEN 拷贝对象所保存的整数值，将其转换成字符串计算长度 调用sdslen函数返回字符串长度 调用sdslen函数返回字符串长度

六、总结

  相比于memecache缓存，Redis支持存储字符串，并对字符串可以做出大量的可处理操作，比如可以对存储的字符串进行一个解析判断是否是数值型的字符串，如果是数值型的字符串可以进行自增自减操作。除此以外还可以通过对字符串子串进行操作比如截取，拼接等日常使用的字符串操作。
  Redis在C字符串上进行了拓展实现自定义的SDS字符串提升程序对于字符串的操作性能以及优化内存分配等。
