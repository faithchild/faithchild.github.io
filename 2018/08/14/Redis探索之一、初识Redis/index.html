Redis探索之一、初识Redis

发表于 2018-08-14
字数统计: 1,092 | 阅读时长 ≈ 4

一、简介

  随着互联网的发展，网络逐渐普及整个社会，能够接触网络上网的人员也越来越多。各个互联网公司的用户量也在日益增长，以前关系型数据库能满足公司的需求，但是慢慢的随着用户量的增长，数据搜索的速度变得越来越慢，这时候就出现了基于内存存储的非关系型数据库。比如Redis、memcache、MongoDB等。
  Redis是一个远程内存数据库，它不仅性能高效而且具备了复制性能以及多样的存储结构。除了这些，它还可以将内存中的数据保存到磁盘中，也可以使用复制功能来拓展读取数据的性能。

二、特点以及功能

  非关系型数据库在小公司或者用户量不大的公司用的并不多，小型的memcache就能解决它们的需求问题，也就是在查询数据之前做了一层缓存。而经常使用的就是关系型数据库MySql，至于Mysql最大的特点就是免费。学习过Mysql的人都知道，在数据库中查询一条数据的麻烦程度，写Sql通过映射关系将表与JAVA Bean联系在一起最终获取想要的结果，而Redis就类似于Map这种key-value键值对，通过key去获取value用法及其方便而且速度高效，这边可以看一下Redis与Mysql、Memcache以及MongoDB之间的区别：

名称 类型 数据存储选项 查询类型 附加功能
Redis 使用内存存储的非关系型数据库 字符串、列表、集合、散列表、有序集合5种 每个类型都有自己的专属命令 发布订阅、持久化、脚本
Memcache 使用内存存储的键值对 键值之间的映射 基本的CURD操作 有缺陷，相比Redis无法持久化
MySql 关系型数据库 很简单的库、表结构 Insert、Select、Delete、Update操作 主从复制、持久化
MongoDB 使用硬盘存储非关系型数据库 可以存储一些大文件 创建、读取、删除等基本操作 支持map-reduce操作、主从、分片等

三、Redis数据结构

Redis中可以存储5种不同的数据，包括String(字符串)、List(列表)、SET(集合)、Hash(散列)和ZSET(有序集合)

结构类型 结构存储的值 读写能力
STRING 可以是字符串、整数或者浮点数 对整个字符串或者字符串的其中一部分执行操作；对整数和浮点数执行自增或者自减的操作
LIST 一个链表，链表上的每个节点都包含了一个字符串 从链表的两端推入或者弹出元素；根据偏移量对链表进行修剪；读取单个或者多个元素；根据值查找或者移除
SET 包含字符串的无序收集器，并且包含的每个字符串都是独一无二、各不相同的 添加、获取、移除单个元素；检查一个元素是否存在与集合中；计算交集、并集、差集；从集合里面随机获取元素
HASH 包含键值对的无序散列表 添加、获取、移除单个键值对；获取所有键值对
ZSET 字符串成员与浮点数分值之间的有序映射，元素的排列顺序由分值的大小决定 添加、获取删除单个元素；根据分值范围或者成员来获取元素

虽然Redis中存储元素的数据结构有这五种，但是对于操这些数据结构的命令却很多。

四、总结

  上面只是列出了Redis一些简单的特性以及优势和特点，而Redis真正的强大在于它能处理分布式的中一些复杂的场景、以及相比memchache这种缓存结构缺少的可持久化的功能、以及单机情况下无法满足而使用集群的方式进行处理。
