二、简单命令以及使用

Redis对于列表操作的命令和JAVA中的双端队列很相似，对于每一个列表可以从表头开始操作也可以从表尾开始操作，具体的命令可以看下面一张表格：

命令 使用格式 行为
LPUSH RPUSH list-key item 从链表左端添加元素，返回的是当前列表的长度
LRANGE LRANGE list-key 0 2 获取列表位置从0到2的元素
LINDEX LINDEX list-key 1 取出列表中在1位置的元素
LPOP LPOP list-key 从列表的左边弹出一个值
LTRIM LTRIM list-key 0 1 对列表进行操作，只保留从0开始到1的元素包含0和1
BLPOP BLPOP list-key [key-name…] timeout 从第一个非空列表中弹出位于最左端的元素，或者在timeout时间内等待
RPOPLPUSH RPOPLPUSH list-key destion-key 从list-key列表中弹出位于最右端的元素，然后将这元素推入目标队列的最左端，并返回这个元素
BRPOPLPUSH BRPOPLPUSH list-key destion-key timeout 在上面的基础上增加了等待的机制

上面的大致涵盖了对列表操作的命令，当然上面大部分的都是从左开始对列表进行操作，而对应的从右开始操作只需要将L改成R即可。从上面的命令看出，Redis相对于Memcache这种简单数据存储的缓存库而言，其在功能上的优势是巨大的。比如RLPOP这些阻塞型的操作就可以将列表当作阻塞队列来使用。

三、总结

  对于Redis的列表操作，比较重要的就是推出推入元素命令这是最基本的添加元素和获取元素的方法。列表的这些命令功能已经很强大，并且注意一点存放在列表中的元素是可以相同的。所以如果希望实现和JAVA中的Set集合一样的原理就需要使用Redis中的第三种数据结构集合。
